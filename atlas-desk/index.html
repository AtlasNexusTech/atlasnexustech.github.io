Streaming P2P

Connexion directe entre vos machines via WebRTC. Aucun serveur intermédiaire ne voit votre écran. Streaming JPEG 15 FPS avec latence minimale.

Sécurisé par défaut

Chiffrement DTLS natif WebRTC. Mot de passe SHA-256 avec challenge-response. ID numérique 9 chiffres ou alias personnalisé.

Transfert fichiers

Envoyez et recevez des fichiers en P2P entre vos machines. Glissez-déposez dans l'app web. Chunks 16KB avec reprise.

Clipboard sync

TURN + Auto-reconnect

Traverse les NATs stricts avec TURN configurable. Reconnexion automatique en cas de perte de connexion. Ne perdez jamais votre session.

App desktop native

Application Tauri avec tray icon et minimize to tray. Pas de navigateur requis. Disponible sur Linux (.deb) et Windows (.exe).

Comment ça marche

Architecture P2P simple. Le signaling server ne fait que mettre en relation — vos données restent privées.

🖥️
Agent

Installé sur le PC distant. Capture l'écran, reçoit les inputs. Go binaire ~10 MB.

🔗
Signaling

Serveur léger qui met en relation agent et client. Relay SDP/ICE uniquement. Go ~5 MB.

🌐
Client

Navigateur web ou app Tauri. Aucune installation nécessaire. Connectez-vous avec un ID ou alias.

# 1. Lancer le signaling server (une fois) ./atlas-desk-signaling # 2. Installer l'agent sur le PC distant ./atlas-desk-agent -alias "PC Bureau" -pass "secret" # → ID: 437 192 805 # 3. Ouvrir le client → entrer "PC Bureau" → Connect
